# 🗳️ ElectIQ — India Election Process Education Assistant

> **PromptWars Virtual Hackathon | Hack2Skill**  
> **Vertical:** Civic Education / Public Service  
> **Built with:** Google Gemini 2.5 Flash + Google Cloud Run + Google Search Grounding

---

## 🎯 Overview

**ElectIQ** is an AI-powered civic education platform that makes the **Indian election process simple, clear, and empowering** for every citizen — especially first-time voters.

It combines a conversational AI assistant (Google Gemini 2.5 Flash), a visual election timeline, an AI quiz engine, a **Gemini-powered fact-checker with Google Search grounding**, and Google Calendar integration — all deployed on **Google Cloud Run**.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🤖 **AI Chat (ElectIQ Assistant)** | Gemini 2.5 Flash-powered conversational assistant with deep India election expertise. **Streaming responses** for real-time token display. |
| 🔍 **AI Fact Checker** | Paste any election claim → Gemini verifies it using **Google Search grounding** and returns a verdict (TRUE/FALSE/MISLEADING) with sources |
| 📅 **Visual Election Timeline** | Step-by-step interactive timeline of the full Indian election lifecycle, filterable by phase (Pre / During / Post) |
| 🧠 **AI-Generated Quiz** | Dynamic 5-question MCQ quiz generated by Gemini on your chosen topic, with explanations for each answer |
| 📆 **Google Calendar Integration** | One-click buttons to save key election dates directly to Google Calendar |
| 📚 **Official Resources** | Curated links to ECI, NVSP, Voter Helpline, PRS, and more |
| 🏠 **Topic Cards** | Quick-access cards that pre-fill the chat with common questions |

## 🏗️ Architecture

```
User Browser (SPA)
     │
     ▼
Google Cloud Run (Flask App)
     │
     ├── /                  → Serves index.html (full SPA frontend)
     ├── /api/chat          → Gemini 2.5 Flash (non-streaming)
     ├── /api/chat/stream   → Gemini 2.5 Flash (SSE streaming)
     ├── /api/quiz          → Gemini 2.5 Flash (structured JSON output)
     ├── /api/fact-check    → Gemini 2.5 Flash + Google Search Grounding
     ├── /api/search        → Google Custom Search API
     └── /api/health        → Health check endpoint
           │
           ├── Google Gemini 2.5 Flash (via google-genai SDK)
           │     └── Google Search Grounding (for fact-checking)
           └── Google Custom Search API
                         │
           User's browser ──► Google Calendar (deep link)
```

**Key design decision:** API keys live server-side on Cloud Run — never exposed to the client browser. The backend supports both **Vertex AI** (production, via service account) and **AI Studio API key** (development).

## 🚀 Deployment — Google Cloud Run

### Prerequisites
- Google Cloud account with billing enabled
- `gcloud` CLI installed and authenticated
- Docker installed

### Quick Deploy

```bash
# Clone repository
git clone https://github.com/OptimistOtaku/ElectIQ.git
cd ElectIQ

# Set your Gemini API key (get from https://aistudio.google.com/)
export GEMINI_API_KEY=your_key_here

# Deploy (uses electiq-civic-edu project by default)
chmod +x deploy.sh
./deploy.sh electiq-civic-edu
```

### Manual Deploy

```bash
# Set project
gcloud config set project electiq-civic-edu
gcloud services enable run.googleapis.com artifactregistry.googleapis.com aiplatform.googleapis.com

# Build and push
docker build -t asia-south1-docker.pkg.dev/electiq-civic-edu/electiq-repo/electiq .
docker push asia-south1-docker.pkg.dev/electiq-civic-edu/electiq-repo/electiq

# Deploy
gcloud run deploy electiq \
  --image asia-south1-docker.pkg.dev/electiq-civic-edu/electiq-repo/electiq \
  --platform managed --region asia-south1 \
  --allow-unauthenticated \
  --set-env-vars "GOOGLE_CLOUD_PROJECT=electiq-civic-edu,GOOGLE_CLOUD_LOCATION=asia-south1"
```

## 🧠 Approach & Logic

### AI Assistant Design
- Gemini 2.5 Flash is given a **strict system prompt** as "ElectIQ" — a non-partisan Indian election expert
- **Streaming responses** via Server-Sent Events (SSE) for real-time token display
- Conversation history is maintained client-side and sent with each request (stateless backend)
- Responses use Markdown formatting, rendered client-side into styled HTML
- Graceful fallback to non-streaming if SSE fails

### Fact Checker (Unique Feature)
- Users paste election claims (e.g., "EVMs can be hacked remotely")
- Gemini 2.5 Flash analyzes the claim with **Google Search grounding** for real-time verification
- Returns structured JSON with: verdict, explanation, official sources, and related facts
- Low temperature (0.3) for factual accuracy
- Grounding metadata extracted for source attribution

### Quiz Generation
- Each quiz is **freshly generated by Gemini** for the chosen topic
- Gemini is prompted to return **structured JSON** for safe parsing
- Questions include explanations to turn wrong answers into learning moments

### Timeline
- Timeline data is **hardcoded with researched, accurate information** about the Indian election process
- Filterable by phase (Pre-election, During, Post-election) to reduce cognitive load
- Each step includes an official timing reference

### Google Calendar Integration
- Uses **Google Calendar RFC 5545 deep links** — no OAuth flow needed
- Each "Add" button opens Google Calendar pre-filled with the event

## 💡 Assumptions Made

1. Users are primarily **Indian citizens** or students learning about Indian elections
2. The app targets **English-speaking** users
3. **Google Custom Search** is optional — the app degrades gracefully without it
4. Calendar links use **Google Calendar** specifically
5. Election dates shown are for **demonstration** — real dates would come from the ECI API

## 🔒 Security Considerations

- ✅ All API keys stored as **server-side environment variables** (never in frontend JS)
- ✅ Backend validates and proxies requests — no direct client-to-AI API calls
- ✅ CORS configured with `flask-cors`
- ✅ Gunicorn used as production WSGI server with gthread workers
- ✅ No user data stored or logged
- ✅ Google Cloud Run provides managed TLS/HTTPS
- ✅ Input validation on all endpoints (min length, required fields)
